## T5 Visibility Problem And Root Causes

The user-reported problem: T5 material replacement was invisible to the eye.
Found three compounding causes:

1. `scripts/make_decal_reference.py` quilted branch treated 0-255 float pixels
   as 0-1 (`np.clip(rgb, 0.0, 1.0) * 255.0`), collapsing the reference panel to
   ~31% luminance. With the embedded T5 config relying on
   `TASK_FINAL_REF_COMPOSITE_SCALE=0.68`, results were either gray (wrong
   color) or pulled back to source (invisible). The same 0-1/0-255 bug made
   the quilted `dark_boost` branch dead code.
2. Texture spatial frequency too high (corduroy ribs ~7 px at 512) — washed
   out by the VAE round-trip.
3. Reference amplitude too low (clipped to ±9%) and further attenuated by
   `luma_preserve=0.62`.

Config trap confirmed: `run_pretty_matrix.sh` defaults to
`RUN_PRETTY_CONFIG_SOURCE=internal` (embedded registry); `scripts/core6_tasks.sh`
applies only in `core6` mode. The embedded T5 had already migrated to a quilted
honeycomb design, which core6_tasks.sh did not reflect.

## Code Changes (all defaults backward-compatible)

- `scripts/make_decal_reference.py`:
  - Fixed quilted 0-255 luma collapse and the dark_boost luma-scale bug.
  - Added `--material-contrast` and `--material-scale` (amplitude / spatial
    period multipliers, default 1.0).
  - Per-pixel highlight cap for bright fabrics (`shade <= 250/luma_px`) so
    texture is carried by shadow grooves instead of saturated white dots.
  - New `waffle` material: thin crisp seam lines, soft raised squares, and
    shadow-region amplitude attenuation.
- `run_edit_sd3.py` + `edit_cli_args.py`: new `--final-chroma-source-scale`,
  `--final-chroma-source-mask`, `--final-chroma-source-mask-blur` final-stage
  projection (keep result luma, blend chroma back to source inside the mask);
  removes VAE iridescent chroma noise and enforces the same-color constraint;
  recorded in metadata as `final_chroma_source_*`.
- `scripts/run_pretty_matrix.sh`: passes material contrast/scale into decal
  generation; `T5_*` env override hooks; `TASK_FINAL_CHROMA_SOURCE_*` wiring in
  the material_panel and localized_decal branches; new canonical task
  `pillow_same_color_cable_knit` in the embedded registry.
- `scripts/core6_tasks.sh`: same new task + `T5_*` hooks for core6 mode.
- `scripts/run_wacv_phase1.sh`: strict/implemented task lists now use
  `pillow_same_color_cable_knit`.

## Generalization Probes And Boundary Fixes (later same day)

Cable-knit recipe verified on three images (white/brown sofa canonical, dark
grey/rattan `pillow_same_color_cable_knit_grey`, white/green armchair
`pillow_same_color_cable_knit_armchair`); summary sheet
`consolidated_2026-06-10/t5_cable_knit_three_images_summary_v4.png`. Issues
found by the user and fixed in order:

1. Texture spill onto surroundings (dark host): bright-host luminance clip
   does not apply to dark pillows; added `--host-mask` semantic intersection
   to `make_decal_reference.py` (decal mask leak 29.4% -> 1.9%).
2. Depth-order violation (knit painted over an occluding table edge): the
   dark-host intersection used dilation; switched to tight erosion for all
   hosts.
3. Pillow silhouette change at a corner (SAM over-segmentation including a
   rattan wedge): added `clip_mask_to_host_color` chroma-consistency guard for
   dark hosts plus `scripts/apply_mask_chroma_guard.py` and the
   `SUPPORT_CHROMA_GUARD` task hook for the semantic mask.
4. Residual leak past the corrected mask: the final edit mask is enforced in
   latent space (64x42 at 512), whose downsampled feather mixes edited content
   over a 2-3 latent-cell band. Added pixel-space
   `--final-outside-restore-scale` (run_edit_sd3.py / edit_cli_args.py;
   `TASK_FINAL_OUTSIDE_RESTORE_SCALE` wiring) restoring outside-mask pixels to
   source at full resolution; corner outside-mask diff 0.0506 -> 0.0038.
   Enabled (1.0) for all cable-knit rows; justified by the task contract
   (material replacement only inside the material mask) and consistent with
   the existing pixel-space chroma projection. Canonical and armchair rows
   were re-run under the uniform final recipe.

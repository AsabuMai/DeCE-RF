## 1. Luminance Gradient Preservation for `u_color`

The color appearance loss now supports an additional source-gradient term:

```text
E_color =
  || UV(current) - UV(target_color) ||^2
  + lambda_luma || Y(current) - Y(source) ||^2
  + lambda_grad || grad(Y(current)) - grad(Y(source)) ||^2
```

Implemented in:

```text
sd3_hrec.py
run_edit_sd3.py
scripts/run_ode_decoupled_edit.sh
tests/test_mask_and_box_helpers.py
```

New parameter:

```text
edit_color_luma_gradient_preserve_scale
```

CLI/env exposure:

```bash
--edit-color-luma-gradient-preserve-scale
EDIT_COLOR_LUMA_GRADIENT_PRESERVE_SCALE
```

Default value:

```text
0.15
```

The term compares horizontal and vertical luminance finite differences inside
the color-edit mask. This is intended to preserve front/body texture and local
edges while the chroma term changes color.

## 2. Eyes Anchor Quality Gate

Auto support for glasses-like edits still uses:

```text
eyes + expanded box
```

but now checks whether the grounded `eyes` anchor is too large. If the anchor
area or bounding-box area is suspiciously large, the semantic mask generator
falls back to:

```text
subject head + upper
```

Implemented in:

```text
scripts/make_semantic_mask.py
scripts/run_semantic_velocity_edit.sh
tests/test_mask_and_box_helpers.py
```

New controls:

```bash
--eyes-anchor-max-area-ratio
--eyes-anchor-max-box-area-ratio
--disable-eyes-anchor-fallback
EYES_ANCHOR_MAX_AREA_RATIO
EYES_ANCHOR_MAX_BOX_AREA_RATIO
```

Default thresholds:

```text
area ratio:     0.16
box area ratio: 0.34
```

Metadata now records:

```text
eyes_anchor_fallback_used
eyes_anchor_area_ratio
eyes_anchor_box_area_ratio
semantic_phrase_after_quality_gate
support_relation_after_quality_gate
```

This directly targets the rabbit/tiger sunglasses failure mode from yesterday,
where animal-eye grounding sometimes covers a large head/body region.

## 5. Eyes Anchor Fallback Validation

Tested the new oversized-eyes quality gate on the rabbit/tiger sunglasses
failure cases.

Initial fallback from `eyes + box` to `head + upper` correctly triggered:

```text
rabbit eyes anchor area: 0.224, box area: 0.455
tiger eyes anchor area:  0.313, box area: 0.415
```

The first fallback band was still too large because it inherited the glasses
rule expansion. The fallback parameters were tightened to:

```text
expand_x = 0.08
expand_y = 0.0
band_ratio = 0.22
```

New fallback mask areas:

```text
rabbit: 0.116
tiger:  0.106
```

Outputs:

```text
outputs/semantic_eyes_fallback_mask_grid_20260506.png
outputs/semantic_eyes_fallback_ode_grid_20260506.png
outputs/semantic_eyes_fallback_tighter_ode_grid_20260506.png
outputs/rabbit_sunglasses_auto_support_fallback_tighter_ode_20260506/result.png
outputs/tiger_sunglasses_auto_support_fallback_tighter_ode_20260506/result.png
```

Readout:

- Tiger benefits from fallback. The wider `0.32` band makes stronger glasses
  but rewrites more; the tighter `0.22` band preserves structure better but
  weakens the glasses.
- Rabbit remains difficult. The tight band prevents the large head/ear rewrite,
  but still does not reliably generate sunglasses on the side-facing eye.
- Current conclusion: the quality gate is useful as a guard against bad eye
  grounding, but side-view animals need a better fallback than generic
  `head + upper`, likely velocity-refined support or a side-eye/head-profile
  rule.

## 6. Rabbit Side-Profile Support Probe

The `head + upper` fallback was still not good enough for the rabbit. It either
failed to create sunglasses or rewrote the head/ears depending on the band
size. This shows the problem is support localization, not just edit strength.

Tested two more support variants:

```text
1. auto structure mask from dark-eye components
2. manual side-profile sunglasses mask around the visible eye
```

Outputs:

```text
outputs/rabbit_sunglasses_structure_mask_ode_20260506/result.png
outputs/rabbit_sunglasses_structure_mask_ode_20260506/masks/structure_glasses_edit_mask.png
outputs/rabbit_sunglasses_profile_mask_ode_20260506/result.png
outputs/rabbit_sunglasses_profile_mask_ode_20260506/masks/profile_sunglasses_mask.png
outputs/rabbit_profile_mask_result_grid_20260506.png
```

Readout:

- The automatic structure detector found dark components, but the selected
  support was too low/right and edited around the mouth/cheek instead of the
  visible eye.
- A manually placed side-profile mask produced a visible black sunglasses-like
  edit on the rabbit eye while avoiding the earlier head/ear rewrite.
- The result is still not polished, but it confirms the correct next direction:
  add an automatic side-profile eye support rule rather than continuing to tune
  generic `head + upper`.

Next implementation target:

```text
profile_eye support:
  single visible eye detector
  one-lens + temple-arm mask
  no second lens unless two-eye confidence is high
```

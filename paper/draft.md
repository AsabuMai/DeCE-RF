# RF h-Edit Draft

This draft should be populated only from complete runs in `outputs/main_matrix`
and metric rows in `experiments/main_metrics.csv`.

## Introduction

Rectified-flow image editing has a useful target-driven velocity field, but a
direct target field can move both the intended edit region and source content
that should remain fixed. This project studies a decoupled ODE formulation that
keeps the source-conditioned RF velocity as the base dynamics, adds an explicit
reconstruction-aware correction, and separates target-seeking edit corrections
from preservation corrections.

The paper claim is intentionally narrow and currently not ready as a broad
success claim: compared with direct target guidance, decoupled RF/ODE editing
improves source faithfulness, but the current outputs only show a clean
positive edit on the cat-crown task. The backpack, yellow-car, and rabbit
tasks should be treated as failures or limitations unless new runs visibly fix
them.

## Related Work

The final related-work section should cover RF/ODE editing methods and

## Method

The implementation-level dynamics are:

```text
xdot_t = v_src + u_rec + u_edit
```

Here `v_src` is the source-conditioned RF velocity. The reconstruction branch
uses the linear RF clean estimate

```text
x0_hat = x_t - t * v_theta(x_t, t)
```

and maps clean-space displacement back to a velocity correction with the
linear-path convention. The editing branch combines target-source velocity
difference, clean-space anchor terms, region terms, optional text/image reward
gradients, surface-reference corrections, and experimental source-reference
Q/K/V injection. These terms are described as energy-inspired surrogate
velocity fields unless a branch explicitly computes an autograd gradient.

The total dynamics are kept fixed while utilities such as mask providers,
surface recolor references, and source-reference injection are treated as
support mechanisms rather than separate core methods.

## Experiments

The submission-oriented matrix is defined in `experiments/main_matrix.md`.
The main tasks are:

- T1: cat crown local headwear insertion.
- T2: backpack blue surface/attribute edit.
- T3: yellow car blue vehicle color edit.
- T4: rabbit side-profile sunglasses robustness case.

Each task is evaluated with:

- M0: base only.
- M1: direct target.
- M2: anchor only.
- M3: decoupled reconstruction.
- M4: full method.

The required seeds are 10, 11, and 12. Each run must have `result.png`,
`stats.json`, `metadata.json`, `command.txt`, and any generated masks.

Metrics are generated by `scripts/evaluate_paper_metrics.py` and include
trajectory summaries, source L1/RMSE, luma SSIM, mask-outside drift, mask-inside
change, runtime, peak GPU memory when available, optional CLIP edit alignment,
optional DINO source similarity, and manual failure flags.

The current 60-row main matrix has complete runtime, peak-memory, CLIP, DINO,
and manual failure annotations. On average, the full method has substantially
lower outside-mask drift than direct target guidance and much higher DINO source
similarity, but visual review shows that this preservation often comes with
weak or incorrect edits. Only the cat-crown full-method rows are currently
marked successful. Backpack-blue rows are hybrid-object failures, yellow-car
rows are color-miss failures, and rabbit-sunglasses rows are localization
failures. This supports at most a preservation-oriented diagnostic result, not
a paper-ready claim of reliable local editing.

The current seed-10 ablation table covers T1-T3 for reconstruction removal,
trajectory removal, attention support, semantic/SAM support, and source V
injection. Attention support is visibly worse on local edits. Source V injection
improves preservation proxies, but qualitative review still shows weak T3 color
editing, so it remains an experimental control rather than the main method.

## Failure Analysis

Known failure modes should remain visible:

- object replacement can become a target-texture or face-overlay failure rather
  than a true object rewrite;
- local accessory insertion can trade placement quality against source
  preservation;
- poor spatial support can produce mask artifacts or suppress the intended
  edit;
- source-reference injection is experimental until its ablation is populated.
- yellow-car color editing is a current weak case: the method preserves the car
  structure but often leaves the body yellow.
- backpack color editing can produce a hybrid object rather than a clean
  surface recolor.

## Limitations

- The method depends on reliable spatial support for local edits.
- The implementation is SD3-specific.
- External baselines require matched conditions before they can support a
  paper claim; current FireFlow/ReFlex artifacts are exploratory only.
- CLIP/DINO metrics are populated, but LPIPS/VLM-style perceptual or
  instruction-following metrics are not yet included.
